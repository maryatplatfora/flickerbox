Common Resource Attributes

Platfora resources share a set of common attributes. Common attributes include identifiers, version information, and object history. This section describes the attributes found on all Platfora resources.

Platfora REST APIs return several common attributes in a return body (GET and POST). These attributes include the following:

Attribute

Description

id

A unique identifier. Platfora assigns this value, callers cannot modify it.

version

The object's version. Platfora assigns this value, callers cannot modify it.

name

An object's unique name. All objects require a name. Callers can read and write names.

history

An array of History objects.

objectPermissions

A read-only structure containing the permission set. These are users or groups that have access to the resource:

  • Owner

  • Create

  • Edit

  • DataAccess

  • Viewer

href

A read-only reference to the object.

A History object is read-only structure with the following values:

Attribute

Description

lifeCycleStage

An enumeration indicating an object's stage. This is one of:

  • modified

  • created

  • deleted

replaces

The version this object replaces.

version

The object's new version. Platfora assigns this value, callers cannot modify it.

time

Time stamp for the stage.

user

A read-only structure identifying the last user to change the object.
