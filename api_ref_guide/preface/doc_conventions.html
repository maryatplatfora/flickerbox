Document Conventions

This documentation uses certain text conventions for language syntax and code examples.

Convention

Usage

Example

$

Command-line prompt - proceeds a command to be entered in a command-line terminal session.

$ ls

$ sudo

Command-line prompt for a command that requires root permissions (commands will be prefixed with sudo).

$ sudo yum install open-jdk-1.7

UPPERCASE

Function names and keywords are shown in all uppercase for readability, but keywords are case-insensitive (can be written in upper or lower case).

SUM(page_views)

italics

Italics indicate a user-supplied argument or variable.

SUM(field_name)

[ ] (square brackets)

Square brackets denote optional syntax items.

CONCAT(string_expression[,...])

...

(elipsis)

An elipsis denotes a syntax item that can be repeated any number of times.

CONCAT(string_expression[,...])
