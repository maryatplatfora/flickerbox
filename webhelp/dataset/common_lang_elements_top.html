Understand Expression Basics

In this section you learn how Platfora's expression language and programmatic query support are related, where you can use them, and which components they share.

Understand the Role of the Expression Language

Platfora's expression language is set of functions and operators that allow you to create an expression. An expression combines functions and operators with field references and literal values to perform a calculation. Expressions can be complicated: 

PARTITION BY UserID
ORDER BY Timestamp
PATTERN (A+)
DEFINE A AS COUNT(A) = 0
OR MINUTES_BETWEEN(Timestamp, LAST(A.Timestamp)) < 30
OUTPUT FIRST(A.Timestamp)

Or an expression can be a very simple reference to a field in some data:

SessionID

Why and Where You Use Expressions

You can use expressions to do sophisticated data processing at several points in Platfora's software. For example, you can use expressions to:

  • filter a data source when creating a dataset
  • create a computed field in a new or existing dataset
  • filter data in a lens
  • create a computed field in a new or existing vizboard

You also use expressions with Platfora's query language. The query language is intended for use in Platfora plugins. You can only query Platfora lenses and then only through Platfora's REST API.

Computed fields, filters, and queries all rely on expressions. Depending on at what point you use them, both computed fields and filters can impact lens builds and data analysis. You'll learn more about these impacts later.

What is the Expression Builder?

The Platfora UI includes an Add Field dialog for building computed fields. The dialog contains an expression builder used to create expressions in data sets and the vizbuilds.



The builder checks your syntax for you; the builder cannot check your logic though! You can also use the builder's help to view the syntax reference.

If you are writing an extension that uses the query language, there is no UI. Instead, you can use the cURL command line, Chrome's Postman extension, or a similar tool to test your query syntax. Refer to the language reference for help writing the SELECT statement for a query.
