About Null and Default Values

If a record does not have a value for a particular field, the field is treated as a null (empty) value. When Platfora builds a lens, null values are substituted with the default field values defined in the dataset.

A Platfora lens is analagous to a summary or roll-up table in a data warehouse. Distinct data values of the selected dimension columns are grouped together in order to calculate an aggregated measure value for each group. If a record has a missing field value, the build process substitutes the default value so that the dataset records can be grouped, summarized, and prepared for visual analysis.

For lenses that include fields from multiple related datasets, Platfora performs an outer join between the fact (or focus) dataset and any referenced datasets included in the lens. This means that rows in the fact dataset are compared to related rows in the referenced datasets. Any row that does not have a corresponding row in the referenced dataset is considered an unjoined reference. The dimension columns for unjoined references are treated as null and replaced with the default values.

If you do not specify your own default values in the dataset, the following default values are used when a lens is built:

Data Type

Default Value

LONG, INTEGER, DOUBLE, FIXED

0

STRING

NULL (as a string)

DATETIME

January 1, 1970 12:00:00:000 GMT
