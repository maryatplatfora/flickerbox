Functions in an Expression

Functions perform common data processing tasks. While not all expressions contain functions, most do. This section describes basic concepts you need to know to use functions.

Inputs and Outputs Functions

Functions take one or more inputs values and returns an output. Input values can be literal values such as 5 or a reference to a field that contains a value. In both cases, the function expects the input value is a particular data type such as STRING or INTEGER. For example, the CONCAT() function combines STRING inputs and outputs a new STRING. You could use the CONCAT() function to concatenate the referenced month, day, and year fields and the STRING literal "/" to create a new string:

CONCAT(month,"/",day,"/",year)

A function's return value may be the same as its input type or it may be an entirely new data type. For example, the TO_DATE() function takes a STRING and creates a DATETIME value. If a function expects a STRING and it gets another data type as input, the function returns an error.

Typically, functions are classified by what data type they take or what purpose they serve. For example, CONCAT() is a string function and TO_DATE() is a conversion function. You'll find a complete list of functions by type in Platfora's Expression and Query Language Reference.

Nesting Functions

Functions can take other functions as arguments. For example, you can use the CONCAT function as an argument to the TO_DATE() function. The result is a DATETIME value.

TO_DATE(CONCAT(month,"/",day,"/",year),"MM/dd/yyyy")

The nested function must return the correct data type. So, because TO_DATE() expects string input and CONCAT() returns a string, the nesting succeeds.

Aggregate Functions versus Row Functions

Most functions process one value from one row at a time. These are called row functions because they operate on one value from a single row at a time. Aggregate functions are a special class of functions. Unlike row functions, aggregate functions process the values from multiple rows together into a single return value. Some examples of row functions are:
  • SUM()
  • MIN()
  • VARIANCE()

For a complete lists of aggregate functions, see the Platfora Expression and Query Language Reference.

Aggregate functions are also special because you use them to define measures. You might recall that measures are always numeric values. This means, the results of an aggregate function is always a numeric data type. Expressions that compute measures are often called measure expressions.

Limitations of Aggregation Functions

Unlike row functions, aggregate functions can only take simple expressions as input (such as field names or literal values). Aggregate functions cannot take row functions as arguments. You also cannot use an aggregate function as input into a row function. You cannot mix aggregate functions and row functions together in one expression.

Finally, while you can build expressions in both the dataset workspace or the vizboards, only the following aggregate functions are allowed in a vizboard expressions:

  • DISTINCT()
  • MIN()
  • MAX()
  • ROLLUP
