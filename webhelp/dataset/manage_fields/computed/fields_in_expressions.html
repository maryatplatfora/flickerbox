Fields in an Expression

Expressions most often operate on the values in fields. To do this, you provide a field name to the function. This section explains how to use field names in expressions.

Referring to Fields in the Current Dataset

When you specify a field name in an expression, if the field name does not contain spaces or special characters, you can simply refer to the field by its name. For example, the following expression sums the values of the sales field:

SUM(sales)

Enclose field names with square brackets ([]) if they contain spaces, special characters, reserved keywords (such as function names), or start with numeric characters. For example:

SUM([Sale Amount])
SUM([2013_data])
SUM([count])

If a field name contains a ] (closing square bracket), you must escape the closing square bracket by doubling it ]]. So if the field name is:

Min([crs_flight_duration])

You enclose the entire field name in square brackets and escape the closing bracket that is part of the actual field name:

[Min([crs_flight_duration]])]>
If you are using the expression builder, it provides the correct escapes for you.
Note: Field is a synonym for dataset column. The documentation uses the word field because that is the terminology used in Platfora's user interface.

Use Dot Notation for Fields in a Referenced Dataset

Your expression might refer to a field in the focus dataset. (Focus dataset is simply the current dataset you are working with.) You also might include a field in a referenced dataset. When including fields in a referenced dataset, must qualify the field name with the proper notation. The convention is reference_name.field_name.

Don't confuse a reference name with the dataset name; they are not the same. When you refer to a dataset, you give the reference a name. Use . (dot) notation to separate the two components. For example consider, the salesTransactions dataset which goes by the salesbyYear reference name. To refer to the Year field of the salesTransactions referenced dataset, you use this notation:

salesbyYear.Year

Just as with field names, you must escape reference names if they contain spaces, special characters, reserved keywords (such as function names), or start with numeric characters. For example:

 [site sessions].sessionLength

Aggregated Functions and Fields in a Referenced Dataset

You cannot directly calculate a measure on a field belonging to a referenced dataset. This means you cannot use these fields as input to the aggregate functions:

AVG([site sessions].sessionLength)

Instead, use a two-step process. First, define theLength computed field whose expression is just the path to the referenced dataset field:

[site sessions].sessionLength

Then, use the computed field as an argument to a second computed filed avg_session containing the aggregate expression:

AVG([site sessions].sessionLength)
