Parse XML Files

The Platfora XML parser supports the selection of a top-level XML element to signify a record or row, and selection of attribute:value or element:value pairs to signify columns or fields (including those in nested elements).

Note: Internally, the Platfora XML parser converts XML elements to JSON format. You can then use JSON expressions and notation to further operate on the parsed data in Platfora.
  1. On the Parse Data step of the dataset workspace, select XML as the File Type in the Parsing Controls panel.
  2. Choose a parser based on the format of your XML files: XML Element per line or XML Document.
  3. (optional) Enter a Filter to the element use as the basis for rows. The parser will make a best guess of the element to use, but you can change this if needed.
  4. Select the columns to include in the dataset definition (you can select either XML attributes or nested child elements).
  5. Only one parsed record is shown at a time, but you can view up to 20 sample records. Columns containing nested elements or attributes are shown in JSON format, not in XML format.
  6. (optional) Use the Data Field Path field to add a column by its path in the document hierarchy. The $. notation is used to specify the root of the XML document element hierarchy.

Parsing Rows from XML

For the XML Element per line format, each line in the file represents a row.

For the XML Document format, the top-level element in an XML document is used by default to signify rows.

If the elements you want to use as rows are contained within a nested element, you can specify a Filter to the element name containing the collection of nested elements you want to use.

For example, in this XML structure, the Filter value would be users (use the collection of child elements contained in the in the users element as the basis for rows): 

<report date="Aug 21, 2012">
    <users>
       <user name="John Smith" email="john@gmail.com"/>
       <user name="Sally Jones" email="sally@yahoo.com"/>
       <user name="Jeff Hamm" email="jhamm@hotmail.com"/>
    </users>
</report>

Or in the example below, you could use the filter report.region.user.address to select the contents of the address element as the basis for rows. Note that the path to the child element is expressed in JSON notation rather than XPath notation. Elements are specified by name. Attributes are specified by their postion within the element (starting with 0). For example, the path to the age attribute would be report.region.user.1.

<report date="Aug 21, 2012">
    <region name="us-west>
      <user name="Georgia" age="42" gender="F">
         <address street="111 Main St." city="Madison" state="IL" zip="35460"/>
         <status id="registered"/>
      </user> 
      <user name="Georgia" age="59" gender="F">
         <address street="45 Pine St." city="Atlanta" state="GA" zip="53291"/>
         <statusupdate type="registered"/>
         <statusupdate type="unsubscribed"/>
      </user> 
    </region>
</report>

Parsing Columns from XML

Once the parser knows the root element to use as rows, the XML element tree is displayed in the Parsing Controls panel. You can add fields contained within XML attributes or nested XML elements by selecting the field name in the XML tree. The field is then added as a dataset column. You can browse through a sample of 20 records in a single file to check for fields to add.

If you unselect a field containing nested XML elements (remove it from the dataset), and later decide to select it again (add it back to the dataset), make sure that Format as JSON string is selected. This will format the contents of the field as JSON rather than a regular string (Platfora stores XML elements as JSON internally).

In some cases, you may want to extract columns from an element one or more levels above the record filter in the XML document structure. To capture upstream elements and attributes as columns, you can add the field by its Data Field Path in the document hierarchy. The $. notation starts the path from the base of the document hierarchy.

  • Supported XML File Formats
    There are two general types of XML files supported by Platfora: XML Element per line files and top-level XML Document files.
