Supported XML File Formats

There are two general types of XML files supported by Platfora: XML Element per line files and top-level XML Document files.

XML Object Per Line Format

The XML Element per line format supports files containing one XML record per line, each record having the same top-level element and structure. For example, here is an XML file where each top-level element represents a user record with one record per line.

<user name="John Smith" email="john@gmail.com"><phone type="mobile" number="123-456-7890"/></user>
<user name="Sally Jones" email="sally@yahoo.com"><phone type="home" number="456-789-1007"/></user>
<user name="Jeff Hamm" email="jhamm@hotmail.com"><phone type="mobile" number="789-123-3456"/></user>

XML Document Format

The XML Document format supports XML document files, where the top-level XML element contains nested XML element records:

<report date="Aug 21, 2012">
    <users>
       <user name="John Smith" email="john@gmail.com"/>
       <user name="Sally Jones" email="sally@yahoo.com"/>
       <user name="Jeff Hamm" email="jhamm@hotmail.com"/>
    </users>
</report>

or a top-level XML element containing a sub-tree of nested XML element records:

<report date="Aug 21, 2012">
    <region name="us-west>
      <user name="Georgia" age="42" gender="F">
         <address street="111 Main St." city="Madison" state="IL" zip="35460"/>
         <status id="registered"/>
      </user> 
      <user name="Georgia" age="59" gender="F">
         <address street="45 Pine St." city="Atlanta" state="GA" zip="53291"/>
         <statusupdate type="registered"/>
         <statusupdate type="unsubscribed"/>
      </user> 
    </region>
</report>

More Complex XML Formats

In some cases, the structure of your XML document might be more complicated. You must specify one node level from the XML element tree to use as the basis for rows. You can, however, still extract columns using a top-level element as well.

As an example, suppose you had the following XML document containing movie review records. You want a row to be created for each reviews record, but still want to retain the value of movie_title and year for each row:

<records>
  <movie title="Friday the 13th" year="1980">
     <reviews>
        <review user="Pam" stars="3">a bit predictable</review>
        <review user="Alice" stars="4">classic slasher flick</review>
     </reviews>
  </movie>
  <movie title="The Exorcist" year="1984">
     <reviews>
        <review user="Jo" stars="5">best horror movie ever</review>
        <review user="Bryn" stars="4">I will never eat pea soup again</review>
        <review user="Sam" stars="4">loved it</review>
     </reviews>
  </movie>
</records>

In the XML parser, the child elements of the movie.reviews element would be used as the record filter. You could then add the movie_title and year attributes by their path as follows:

$.movie.0
$.movie.1

The $. notation starts the path from the base of the document element hierarchy. Elements are specified by name. Attributes are specified by their postional number starting with 0.
