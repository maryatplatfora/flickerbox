Configure LZO Compression for Platfora

This section describes how to support for LZO compressed files to Platfora. Hadoop expects splittable compression formats. For that reason, the LZO compressed file format requires both the base LZO installation and the Hadoop-LZO GitHub project.

Platfora Support provides a pre-built package of LZO with Hadoop-LZO. This pre-built version is compatible with CentOS and Redhat. If you are running Unbuntu, you will need to create your own package.
  1. Follow your Hadoop distribution's instructions for adding the LZO codec.
  2. Log into the Platfora server node as the platfora user.
  3. Make sure you have a $PLATFORA_DATA_DIR/extlib and $PLATFORA_DATA_DIR/extlib/native directory. If you don't create, them.
  4. Download the Hadoop-LZO package from Platfora Support.
  5. Extract the contents of the file. 
    $ tar -xvf filename.tar
  6. Copy the /native/libgpl*.so files to $PLATFORA_DATA_DIR/extlib/native directory. 
    $ cp /tmp/libgplcompression/native/* $PLATFORA_DATA_DIR/extlib/native
  7. Ensure the libraries have a mask value of 755. 
    $ chmod 755 libgpl*
  8. Copy the hadoop-lzo-0.4.15.jar file to $PLATFORA_DATA_DIR/extlib directory. 
    $ cp /tmp/libgplcompression/hadoop-lzo-0.4.15.jar $PLATFORA_DATA_DIR/extlib
  9. Install the liblzo* libraries into your /usr/lib directory. 
    $ sudo cp /tmp/libgplcompression/usr/lib/* /usr/lib
  10. Update your local Hadoop configuration with the LZO codec changes from your Hadoop distribution.
    For example, the core-site.xml file contains the io.compression.codec.lzo.class property key.
  11. Restart Platfora.
