Write a Filter Expression

Lens filters limit the number of records included in a lens. You can define a lens filter on any dimension field included in your lens. Since filters restrict the available data, use them thoughtfully as they can limit your analysis scope.

Understand Filter Expression Basics

Filter expressions are always Boolean expressions. This means filters use comparison operators or logical operators.

Filter expressions (also known as Boolean expressions) are expressions that evaluate to true or false. You can use filter expressions to limit the number of records included in a lens, or to define the conditions of a CASE expression, DEFINE clause, or WHERE clause.

You can only filter on dimension data. When you filter a field, you reduce the number of rows pulled into your analysis. You can only define one filter per field.

The field name serves as the first argument of the expression, followed by a comparison or logical operator, and then the comparison value. The comparison value must be of the same data type as the selected field you are filtering on.

When you are writing filters, you use the Define filter dialog.
