Add a Dataset

Platfora datasets define how to parse, cleanse, transform, and process Hadoop source data. The sample flights data comes with four predefined datasets: Flights, Monthly Carrier Activity, Airports, and Airline Carriers. You can examine these datasets to see different examples of computed fields, measures, and references. In this exercise, you will add an additional dataset for Planes.

The dataset workspace is where you define or edit a dataset. A dataset is a metadata description of where the source data is located in Hadoop, how it is structured, and how it should be processed. The dataset workspace is divided into six steps. You do not have to do the steps in order - you can move back and forth between steps as you work on the dataset definition.

Step 1: Select Data

The first step in creating a dataset is selecting some source data that resides in Hadoop. This is accomplished by choosing files from a data source. The tutorial data is added to the default Uploads data source.

  1. Go to the Data Catalog, and select the Datasets tab.
  2. Click Add Dataset.
  3. In the Source List, select the Uploads data source.
  4. Select the planes.csv file.
  5. Click Continue.

Step 2: Parse Data

The Parse Data step of the dataset workspace is where you describe your source data by parsing it into rows (records) and columns (fields). For Platfora to be able to parse the source data, each record must have a similar data structure.

Platfora takes a sampling of the first 20 lines in the source file. The Wrangled tab shows the data with the parsing logic applied. The Raw tab shows the raw lines as they are in the source data.

The tutorial data files are in comma-separated values (csv) file format, which uses the Delimited parser.

  1. Make sure the Delimited parser is selected. The planes.csv file uses the default row and column delimiters.
  2. Select Raw file contains header. This uses the first line for the column names rather than as a data record.
  3. Click Continue.

Step 3: Manage Fields

The Manage Fields step is where you verify, cleanse, and transform the raw data fields. The result is data that users can browse and select in the Platfora data catalog. The Managed Fields step is where you do the majority of the dataset definition work.

A dataset can contain base fields and computed fields. Base fields come directly from the source data. Computed fields are fields that you transformation or process.

During the Parse Data step, Platfora automatically transformations date fields it recognizes. For example, the planes.csv source contains an xx_date_acquired column. The column contains simple string dates such as 10/25/10. When you bring the source into your dataset, Platfora automatically transforms this string field into DATETIME format. The new DATETIME value references two Platfora built-in datasets TIME and DATE (more about these later). The original string date had no time, so Platfora use midnight for the time on all the fields.

Platfora also automatically adds a Total Records field to your data. Total Records is a computed field that uses an aggregation function to compute a measure. The measure is the count of records in the dataset. You'll learn more about functions later.

  1. Hide the base fields that are prefixed with xx_ (xx_carrier, xx_lease_status, xx_in_operation, xx_acquired_date). This data will be processed using computed fields or references.

Step 4: Create References

A reference allows two datasets to be joined together on fields that they share in common. A reference creates a link in one dataset to the primary key of another dataset.

The built-in Date and Time dimension datasets have a hierarchy that allows users to explore dates at different granularities. In some cases, these references may not be relevant (for example, if there is no time component in your data). You can hide or rename these auto-generated references, but you cannot delete them.

  1. Select the auto-generated reference to Date.
  2. Change the reference name to Acquired Date and click Update.
  3. Click Add New to define a new reference.
  4. In the Define References panel, select Airline Carriers for the Focus Dataset, select xx_carrier_code for the Foreign Key, and enter Carrier as the reference name.
  5. Click Add to add it to the dataset.
  6. Select Hidden on the auto-generated reference to Time.
  7. Click Continue.

Step 5: Define Key

A key is single field (or combination of fields) that uniquely identifies a row in a dataset, similar to a primary key in a relational database. A dataset must have a key defined to be the target of a reference, the base dataset of a segment, or the focus of an event series lens. Also, the key field(s) used to join two datasets must be of the same data type.

  1. Select Include in Key for the xx_carrier and the Tail Number field (this dataset has a compound key).
  2. Click Continue.

Step 6: Finish and Save

The last step in defining a dataset is making sure the name is what you want to appear in the Platfora data catalog. You cannot change the dataset name after it has been saved, so choose the name carefully! You may also want to add a description to help users understand the data in the dataset.

  1. Change the dataset name to Planes.
  2. Enter a description for the dataset.
  3. Click Finish.

Step 7: Edit Dataset Permissions

By default, a dataset is owned by the user that created it. To make the Planes dataset fully available to other users, we will assign ownership to the Everyone group.

  1. Choose Permissions from the Planes dataset action menu.
  2. Find the Everyone group, click move to..., and select Own.
  3. Click OK.
