Model References and Events

A reference allows two datasets to be joined together on fields that they share in common. A reference creates a link in one dataset to the primary key of another dataset. An event is similar to a reference, except that the direction of the reference link is reversed. An event is created in a dataset that has a primary key and points to a foreign key field in another dataset.

In this exercise, we will edit the Flights dataset and add a reference to the Planes dataset. This will allow us to choose additional dimension fields about planes when we build an aggregate lens from the Flights dataset.

Once the reference to Planes has been defined, we can then add an event reference in Planes that points to a specific event in Flights. Modeling events is only necessary if you want to build event series lenses.

For more information about the tutorial datasets and their relationships, see About the Tutorial Sample Data.

Step 1: Model a Reference

References are created in a fact dataset (Flights in this example), and point to a dimension dataset (Planes in this example). Creating a reference allows the datasets to be joined when building lenses, similar to a foreign key to primary key relationship in a relational database.

The focus dataset must have a key defined before you can create a reference to it. Also the foreign key in the fact dataset must be of the same data type as the key field(s) in the dimension dataset.

  1. Go to the Data Catalog, find the Flights dataset, and choose Edit from the dataset action menu.
  2. Go to the Create References step of the dataset workspace.
  3. Choose Planes for the Focus Dataset.
  4. Choose Carrier Code and Tail Number as the Foreign Keys.
  5. Name this Reference Plane and click Add.
  6. Click Save and Exit.

Step 2: Model an Event Reference

Event references are created in a dimension dataset (Planes in this example) and point to a fact dataset (Flights in this example). Before you can define an event, you must create regular references first.

Modeling event references in a dimension dataset allows you to define event series lenses from the dataset. Since event series lenses order event records by time, event references also require you to choose a datetime type field.

  1. Go to the Data Catalog, find the Planes dataset, and choose Edit from the dataset action menu.
  2. Go to the Create References step of the dataset workspace.
  3. Click Add Event.
  4. Enter Flight Departure as the Event Name.
  5. Choose depart_datetime as the Ordering Field. Event records are always ordered by a datetime type field in an event series lens.
  6. Click Add Event.
  7. Click Save and Exit.
