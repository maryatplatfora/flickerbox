Edit and Rebuild a Lens

The sample flights data comes with one predefined aggregate lens, October 2012 Flight Data, which has a selection of measure and dimension fields chosen from the perspective of the Flights dataset. In this exercise, we will edit the October 2012 Flight Data lens definition, change some of its field selections, add a lens filter, and then rebuild the lens to get the newly requested data from Hadoop.

Step 1: Edit and Rebuild a Lens

The October 2012 Flight Data lens has already been defined and built once, but now we want to modify it to include additional fields. We will also add additional measures to the lens by choosing quick measure aggregations. Lastly, we will add a lens filter to limit the records included in the lens.

  1. Go to the Lenses tab in the Data Catalog.
  2. Edit the October 2012 Flight Data lens.
  3. Find the Plane reference and click Add All. This will add all dimension fields from the Planes dataset to the lens.
  4. Define a lens filter on the Source File YYYY-MM field using the filter expression: ="2012-10".
  5. Find the Taxi In Minutes and Taxi Out Minutes fields and click + to add them to the lens.
  6. Edit the quick field selections for the Taxi In Minutes and Taxi Out Minutes fields and deselect Original Value. This will include these fields in the lens as measures but not as dimensions.
  7. Click Build/Update to save the lens definition and rebuild the lens.
  8. Click Confirm.

Step 2: Check Lens Build Progress

  1. Go to the Activities & Lenses section of theSystem page.
  2. Monitor the progress of the October 2012 Flight Data lens build.

  3. When the lens build completes, you may see a warning about NULL foreign keys. Don't worry, NULL foreign keys are expected for this data. Since departure and arrival dates and times are always NULL for cancelled flights, it is expected that some dates and times will be empty. The references to the built-in Date and Time dimension datasets are what is causing this warning.
