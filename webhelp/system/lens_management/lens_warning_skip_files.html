Skipped Files

A Skipped Files warning means that some source data files were skipped during lens build processing, and the lens may not contain all of the data you were expecting.

Lens builds skip over compressed source data files when the file is corrupt or when Hadoop does not recognize the compression format. To find out what source files were not processed, check your MapReduce task logs in Hadoop.

Configure the Skipped File Threshold

By default, Platfora skips all corrupt compressed files when building a lens. However, system administrators can configure Platfora to fail lens builds when it encounters a certain number of corrupt compressed source files. Set the platfora.max.corrupt.files.to.skip server configuration property to the acceptable number of skipped files. Lens builds that exceed this limit would then fail instead of suceeding with a warning.
