Check for Final Hadoop Parameters

Hadoop may ignore your Platfora-generated job parameters causing your lens builds to fail. This happens if the Hadoop configuration overrides what is set in your local Platfora installation.

The Platfora installation requires you to create a Hadoop configuration directory on each Platfora node. You use this local configuration to set Hadoop parameter values for Platfora requests. When requesting a MapReduce job, Platfora passes the local values to the Hadoop cluster. The expectation is that Hadoop uses the values passed by Platfora to run the job.

However, within a Hadoop cluster, Hadoop administrators can declare Hadoop configuration parameters as final. Once an administrator declares a value final, clients such as Platfora cannot alter that value. If the Hadoop configuration contains a final parameter, Hadoop ignores any value Platfora might pass for that same parameter.

Keep final parameter interactions in mind when troubleshooting Hadoop services. Ensure that any values you set within Platfora's node configuration are not marked as final parameter settings in Hadoop's own configuration.
