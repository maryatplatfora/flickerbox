Avoid Killed Tasks by the CapacityScheduler

The CapacityScheduler allows you to schedule tasks based on a job's memory requirements, the amount of available RAM, and virtual memory on a TaskTracker node. If your Hadoop distribution is using the CapacityScheduler, your lens builds can exceed the scheduler limits.

Most Hadoop distributions have the CapacityScheduler feature disabled and others enable it by default. Consult your distribution documentation to find what your distribution does.

The following properties control the CapacityScheduler memory. These properties are not Platfora-settable properties. These are set on your Hadoop TaskTracker nodes by your Hadoop administrator.

  • mapred.cluster.max.map.memory.mb
  • mapred.cluster.max.reduce.memory.mb
  • mapred.cluster.map.memory.mb
  • mapred.cluster.reduce.memory.mb
  • mapred.job.map.memory.mb
  • mapred.job.reduce.memory.mb

Make sure the memory requested by Platfora jobs does not exceed the limits set on the scheduler. Platfora nodes request jobs using the mapred.java.child.opts value in the local mapred-conf.xml. If a job consumes more memory than the CapacityScheduler limits allow, then its tasks are killed.
