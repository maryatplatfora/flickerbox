Capture Hadoop Logs for Failed Lens Builds

When Platfora builds a lens, it submits MapReduce jobs to Hadoop for processing. When a MapReduce job fails, information from the Hadoop logs is key to diagnosing what went wrong. System administrators can configure the amount of diagnostic data Platfora server logs include from the Hadoop TaskTracker logs. You might want to increase the amount of diagnostic data when troubleshooting lens build failures.

The following Platfora server configuration properties allow administrators to configure the amount of Hadoop TaskTracker information captured in the Platfora log files:

Property

Default

Description

platfora.debug.failed.task.log.num

5

If a lens build job fails, the number of failed Hadoop task logs to include in the Platfora server log. This applies to both map and reduce tasks. If you want to capture all failed task logs, ensure that this number is equal to (or greater than) the number of nodes * the number of map + reduce task slots in your Hadoop cluster.

The default is 5.

platfora.debug.mapper.task.log.num

1

The number of map task logs to include in the Platfora server log when a lens build job succeeds.

platfora.debug.killed.task.log.num

3

If a lens build job fails, the number of killed Hadoop task logs to include in the Platfora server log. This applies to both map and reduce tasks. If you want to capture all killed task logs, ensure that this number is equal to (or greater than) the number of nodes in your Hadoop cluster.

platfora.debug.reducer.task.log.num

1

The number of reduce task logs to include in the Platfora server log when a lens build job succeeds.
Note: Including task logs will increase the size of the Platfora server logs, especially if you have a lot of lens build failures. Platfora recommends only increasing the amount of task logs captured when troubleshooting lens build issues.
