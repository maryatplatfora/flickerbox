Where to Look When a Lens Build Fails

Troubleshooting a lens build failure is an iterative process. Start by looking in Platfora for information, and then moving to Hadoop for additional information if necessary.

Platfora System Page

When a lens build fails, the first step typically is to view which job in the lens build series failed. Check there first to see if there is a useful message. The Unsuccessful tab on the System > Activities & Lenses page lists all canceled and failed lens builds. Expand the failed lens and then click the informational icon under the Msg column.

Platfora Server Log

Search in the Platfora server log by timestamp or error message to view more information. The Platfora master server log file is located at $PLATFORA_DATA_DIR/logs/platfora-server.log. You can also view the log file on each node on the System > Resources page.

Hadoop JobTracker Log

Sometimes you need to view the Hadoop logs for more details about what happened in Hadoop than the System page provides. The Platfora System > Activities & Lenses page links to the JobTracker URL for a particular lens build step. Expand the lens build and click one of the steps. Platfora displays a link to the JobTracker URL so you can see more details from Hadoop.

Note: To view the JobTracker Log for Amazon EMR, the Platfora server configuration property platfora.emr.termination.protected must be enabled (set to true). By default, this property is disabled. When enabled, EMR instances are not terminated after a lens build completes. Note that all instances must be manually terminated in the AWS console. Platfora recommends enabling this property as needed to troubleshoot lens build issues. You can also configure Platfora to copy the EMR job flow log files to a directory on S3 by specifying an S3 native URI in the platfora.emr.log.uri configuration property. This allows you to access the job flow log files at any time for troubleshooting purposes.
Note: You might need to contact your Hadoop administrator to access the JobTracker page. However, Platfora copies the JobTracker configuration file XML (Job File) from Hadoop and stores it on the Platfora server. Whether or not you have access to the JobTracker URL, you can view this XML file stored in Platfora by clicking View job XML Output. Right-click the link to view it in a separate browser tab or window.

From the Hadoop JobTracker page, you can see different details about the job. If the job status is Failed, first confirm that the job setup was successful. You can view the configuration (Job File) used for this job to note the different configuration property settings used. If job setup was successful, you can drill down to view details for each TaskTracker task. You can click the various links that provide more insight into the map tasks, reduce tasks, and the failed and killed map and reduce tasks.
