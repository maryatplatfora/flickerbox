Troubleshoot Hadoop Configuration Problems

This section describes how to adjust certain Hadoop-controlled parameters to avoid some common MapReduce errors in Platfora lens builds. In the context of Hadoop, Platfora is considered a client. As a client, Platfora can request or set some parameters on a MapReduce job. Other parameters Hadoop controls.
  • Check for Final Hadoop Parameters
    Hadoop may ignore your Platfora-generated job parameters causing your lens builds to fail.
  • Avoid MapReduce Out-of-Memory Errors
    Better tuning of the TaskTracker task properties and Java virtual memory (JVM) settings can fix these errors.
  • Avoid Killed Tasks by the CapacityScheduler
    If your Hadoop distribution is using the CapacityScheduler, your lens builds can exceed the scheduler limits.
  • Avoiding Dictionary Errors in Lens Builds
    This section describes how to configure Platfora to avoid dictionary errors resulting from modifications to HDFS and S3 data sources.
