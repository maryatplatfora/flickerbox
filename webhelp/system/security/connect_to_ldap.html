Configuring LDAP Access

You can configure Platfora to access an external LDAP directory server to authenticate users. Platfora currently supports authenticating users in Microsoft Active Directory only.

To enable Platfora to authenticate users in LDAP, you must supply the connection information to the LDAP server.

  1. Go to the System page.
  2. Go to the Users tab.
  3. Click Configure LDAP.
  4. Make sure the Enable LDAP check box is selected.
  5. Enter the connection information for the LDAP server.
    Field Description

    LDAP Server

    The external DNS name or IP address of the LDAP server.

    Port

    The port that the LDAP server is listening on for client connections.

    Use default port

    Select if the LDAP server is listening on the default port numbers. The default LDAP port is 389. The default LDAPS port is 636.

    Use SSL to connect

    Select to encrypt connections between Platfora and the LDAP server. The Platfora server must have SSL enabled in order to use this option. Check your browser address bar to see if HTTPS is being used. If not, you will have to run setup on the Platfora master server to configure it for SSL before you can enable this option. See the Installation Guide for details.

    Search Base

    Specifies the distinguished name (DN) of the location in the LDAP directory tree from which to search for user and group entries.
  6. Enter the Group Attributes used to identify group entries in the LDAP directory.
    Field Description

    Group Name

    This is the name of the LDAP attribute that specifies the group name. The default attribute for Active Directory is name.

    Group Member for Users

    This is the name of the LDAP attribute that specifies the members of a group. You can specify any LDAP attribute to determine membership for a user, but the attribute specified here must store values as either a Distinguished Name, LDAP user ID, or LDAP user name.

    The default attribute for Active Directory is member.

    Group Member for Groups

    This is the name of the LDAP attribute that specifies the members of a group. You can specify any LDAP attribute to determine membership for a group, but the attribute specified here must store values as either a Distinguished Name or LDAP group name.

    The default attribute for Active Directory is member.

    Group Member Of

    This is the name of the LDAP attribute that specifies the parent group of a group. You can specify any LDAP attribute to determine membership for a group, but the attribute specified here must store values as either a Distinguished Name or LDAP group name.

    This field is optional. When a value is specified here and in the Group Member for Users and Group Member for Groups fields, Platfora uses the attribute specified here to determine membership and ignores anything configured in the other two fields.

    Group Object Classes

    These are the LDAP object classes used to define a group. The default object classes for Active Directory groups are top, group.

  7. Enter the User Attributes of the LDAP user account that will be used to bind to the LDAP directory.
    Field Description

    User Name

    This is the name of the LDAP attribute that specifies the user name. The default attribute for Active Directory is name.

    User ID

    This is the name of the LDAP attribute that specifies the unique user ID. The default attribute for Active Directory is sAMAccountName.

    User Member Of

    This is the name of the LDAP attribute that specifies the group of a user. This is optional. You can specify any LDAP attribute to determine membership for a user, but the attribute specified here must store values as either a Distinguished Name, LDAP user ID, or LDAP user name.

    User Object Classes

    These are the LDAP object classes used to define a user. The default object classes for Active Directory groups are user, top, person, organizationalPerson.
  8. Enter the types of values stored in the membership fields.
    Field Description

    Platfora Group Member Field

    The type of value stored in the configured group membership fields, either Group Member for Groups or Group Member Of. The membership value can be either an LDAP Distinguished Name, LDAP group name, or the Platfora group name.

    For example, when the Group Member for Groups field specifies member or the Group Member Of field specifies memberOf, choose LDAP Distinguished Name, and when the Group Member for Groups field specifies memberUid, choose LDAP Group Name.

    Platfora User Member Field

    The type of value stored in the configured user membership fields, either Group Member for Users or User Member Of. The membership value can be either an LDAP Distinguished Name, LDAP user ID, or LDAP user name.

    For example, when the Group Member for Users field uses member or the User Member Of field uses memberOf, choose LDAP Distinguished Name.
  9. Enter the User Attributes of the LDAP administrative user account that will be used to bind to the LDAP directory server.
    Field Description

    LDAP Admin User DN

    The distinguished name (DN) of the administrative user that Platfora should use to bind to the LDAP directory server.

    Password

    The password for the administrative user.

    Save this login info

    If selected, these LDAP credentials will be saved in the Platfora metadata catalog database, which currently does not store passwords in encrypted format. If you choose not the save the login credentials, then you must configure your LDAP directory server to allow anonymous access (so Platfora can bind to the LDAP server without supplying login credentials).
  10. Click Test Connection to verify that Platfora can connect to the LDAP server.
    Platfora will use the LDAP server connection information and the given user DN and password to see if it can bind to the LDAP server and authenticate the user. OK means that LDAP authentication is successfully enabled. If the test connection fails, make sure that you can actually ping the LDAP server over the network and that the search DN and user credentials are correct in the LDAP directory tree.
  11. Click Save to save the LDAP connection information and enable LDAP authentication.
