Understand Object Permissions

Object access permissions control what users can do with a Platfora object (a data source, dataset, lens, or vizboard). Object access and data access are completely independent of each other in Platfora. A user must be granted access to both in order to work with an object and the data it contains.

For example, if a Mary has View Only object access to a vizboard, that means she can open the vizboard and navigate its pages. However, if the visualizations within the vizboard were created with source data that Mary is not allowed to see, those visualizations will be disabled for Mary (grayed out).

The following permissions can be granted on Platfora objects:

Object Permissions
Data Source

The ability to create, update or delete data sources is restricted to System Administrators only

Define Datasets – The ability to define datasets based on files residing in the data source location.

Manage Permissions – Define Datasets plus the ability to grant Define Datasets access to others.
Dataset

Own – The ability to edit and delete the dataset, and grant permissions to others. The dataset creator is the default owner.

Edit – The ability to edit the dataset definition and define/build lenses.

Define Lens – The ability to see the Data Catalog view only of the dataset (cannot see raw data) and define/build lenses from the dataset.

Lens

Own – The ability to edit and delete the lens definition, unbuild a lens, and grant permissions to others. The lens creator is the default owner.

Edit – The ability to edit the lens definition and update/rebuild the lens.
Vizboard

Own – The ability to edit and delete the vizboard, and grant permissions to others. The vizboard creator is the default owner.

Edit – The ability to edit the vizboard and its contained pages and visualizations. Can save or duplicate the vizboard.

View Only – The ability to view the vizboard and its contained pages and visualizations. Can adjust filters and page layout controls in the current session only (cannot save changes).
