Understand Data Access Permissions

Data access is a binary control in Platfora. A user is either authorized to see the data or they are not. Once a user has data access, they will be able to see the data inside any object (dataset, lens, or vizboard) to which they also have object access permissions.

System administrators can control data access at the following levels:

  • Data source. Granting data access on a data source applies to any datasets derived from that data source, as well as any lenses or visualizations derived from those datasets. This is the default.

  • Dataset. Controlling data access on a per-dataset basis allows administrators more fine-grained control by granting a user data access to only the source files used in a particular dataset instead of all files in a data source.

  • HDFS file permissions. Let the HDFS file permissions enforce which LDAP users have data access to each data source and dataset by enabling HDFS Delegated Authorization. For details on enabling and using this feature, see Understand HDFS Delegated Authorization.

By default, data access is controlled at the data source level only. To control data access at the dataset level, too, a system administrator must configure Platfora to enforce data access permission at both the data source and dataset level. For details on how to do this, see Configure Data Access Enforcement.

By default, users with the role of System Administrator in Super Admin mode bypass all data access controls. However, when HDFS Delegated Authorization is enabled, data access permissions for super administrators are still enforced by HDFS file permissions.
