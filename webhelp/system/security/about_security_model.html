Manage Access and Permissions

Platfora provides role-based access control (RBAC) to configure what users can do in the Platfora application, what data they can see, and what objects they can access. Access can be controlled at both broad and detail levels.

  • The Three-pronged Approach
  • Understand Platfora System Roles
    Platfora currently has five system roles that can be assigned to a user (or group). These roles control the actions that users can do in the Platfora application. In addition to these roles, Platfora includes a Super Admin mode that allows system administrators to perform all actions.
  • Understand Object Permissions
    Object access permissions control what users can do with a Platfora object (a data source, dataset, lens, or vizboard). Object access and data access are completely independent of each other in Platfora. A user must be granted access to both in order to work with an object and the data it contains.
  • Configure Data Access
    Data access grants the ability to view data exposed by a data source. Users must have data access to a source in order to use datasets, lenses, or vizboards that use data from the source. Users have access to data in a source by being granted data access to a configured data source or dataset. Platfora can be configured to enforce data access permissions at different levels.
