Configure LDAP User Authentication

Platfora can be configured to use an external LDAP directory service to authenticate users. With LDAP enabled, users log in to Platfora using their familiar LDAP identity and credentials.

When you enable LDAP authentication in Platfora, all user identity information is then maintained in LDAP, not in the local Platfora user accounts. System administrators can map LDAP groups or individual users into Platfora. The local Platfora accounts are then used solely for the purpose of assigning Platfora system roles and permissions.

  • Configuring LDAP Access
    You can configure Platfora to access an external LDAP directory server to authenticate users. Platfora currently supports authenticating users in Microsoft Active Directory only.
  • Mapping LDAP Users into Platfora
    There are two ways to map an LDAP user into Platfora. One way is to map an LDAP group that the user is a member of. The other way is to map an individual LDAP user into Platfora.
  • Mapping LDAP Groups into Platfora
    You can add an LDAP group to Platfora by mapping in its distinguished name. LDAP users that are members of the group will then be able to access Platfora. When an LDAP user logs in to Platfora, they are automatically mapped into Platfora as long as they have group access.
