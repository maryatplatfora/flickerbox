Installation Variables and Directories

This section describes the directories, important files, and environment variables in a Platfora installation .

Environment Variables

The Platfora installation process asks you to create several variables in a node's environment. This variables are the following:

Variable Name

Description

PLATFORA_HOME

Location of the Platfora installation files.

PLATFORA_DATA_DIR

Storage for Platfora's data and work files.

PLATFORA_CONF_DIR

Local directory where Platfora stores its configuration files.

Directories and Key Files in PLATFORA_HOME

This section describes the key files and folders in the installation directory.

Directory or File

Description

bin

Contains the management utilities. You should only run these utilities from the master node.

client

Contains files that support the REST client. You can use the client to create and manipulate Platfora objects via HTTP requests to the server.

The directory contains a few example client tools that you can either use directly or as examples for writing your own Platfora client programs.

conf

Contains default configuration files used to populate PLATFORA_CONF_DIR.

docs

Contains the documentation files and the license for open source code included with the projects.

server

Contains the server libraries.

tools

Contains files that support user defined functions (UDF).

setup.py

Initializes a new Platfora instance or upgrades an existing one. You can also use this utility to reset bootstrap system configuration properties.

Working with the PLATFORA_DATA_DIR

The PLATFORA_DATA_DIR should have enough disk space to accommodate the Platfora server log files, the PostgreSQL metadata catalog database, and materialized lens data. During installation, the setup.py utility ensures that this directory has at least twice the amount of space as system memory. The PLATFORA_DATA_DIR must be writable by the platfora system user.

The installation process asks you for the data directory location and stores the value in the platfora.data.dir property.
