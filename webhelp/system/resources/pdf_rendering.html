Managing PDF Rendering

Platfora users can render a vizboard to PDF. Then, users can download and email the PDF as needed. This section describes how to manage vizboard-to-PDF rendering in the Platfora system.

You can manage whether or not users have the choice to render to PDF. Rendering a vizboard launches a build PDF render process. You can also configure the rendering process. The following Advanced properties manage PDF rendering:

Property Description
platfora.rendering.enabled Enables or disables rendering options on the Vizboard. The default is true.
platfora.rendering.job.timeout Sets the timeout for PDF Rendering jobs. The default is 300 seconds.
platfora.rendering.job.artifact.maxAge Specifies how many days to retain job artifacts. The default is 1 day.
platfora.rendering.maxJobs Sets the maximum number of rendering jobs to queue. The default is 3.

Configure PDF Rendering Properties

  1. Log into Platfora as a user with administrative rights.
  2. Select the System page.
  3. Choose Global Settings.

    The Global Settings dialog displays.

  4. Choose Advanced from the left-hand menu.
  5. Expand the rendering section.
  6. Set the configuration values.
  7. Apply Settings to close the dialog.
  8. Restart the Platfora master node.

Manage Rendering Jobs

From the System page, the Activities & Lenses view shows you data about PDF rendering jobs. Administrations can view In Progress, Unsuccessful, and All activities from all users, including PDF rendering.

In checking for Unsuccessful rendering jobs, you may notice that they frequently exceed the timeout. Review the job detail to see which phase in the render process is taking the most time. For example, if jobs are spending no time waiting for a slot but failing in the later steps, you could increase the platfora.rendering.job.timeout value.
