Using the Platfora Logs

You can review activity on the Platfora system using the logs in the $PLATFORA_DATA_DIR/logs directory. This directory exists on every master and worker node.

Standard Log Files

Platfora users Apache's log4j Service to define and configure its log files. The master node's $PLATFORA_HOME/conf directory contains the log configuration files. The master node supports the following log files:

Log Description
platfora-server.log The primary log file. Contains messages from the Platfora system. Lens build failures, program exceptions, and other key messages appear in this log. For example, you would use this log to troubleshoot connection issues.
access.YYYY_MM_DD.log A daily log of the REST requests made to the server. This log rolls over every day.

platfora-events.log

 Contains query event messages. Each message is a JSON formatted string with the following format: 
{
  "type": "Message Type",
  "actor": "akka://Platfora/user/Query_1403553496123_2010/DictionaryFromMeta_7",
  "target": "akka://Platfora/user/Query_1403553496123_2010",
  "message": "Message",
  "time": 1403568008505,
  "user": "username",
  "node": "0"
 }
platfora-cluster.log Records the Platfora the node availability.
platfora-access-audit.log Contains messages from object access system.
platforaWatchdog.log Contains messages from the Platfora server service.
pg.log Contains messages from Platfora's Postgres database.
platfora-agent.log Contains messages from EMR clients.
hadoop-check.log Contains message from Platfora's hadoop-check utility. You run this utility after installing Platfora or at the request of Platfora Support.

Unsupported Log Files

The following log files are not supported. Platfora may remove them or change their formats at any time without notice. You should not build upon or rely on these files.

Log Description
proton-usage.log An internal log file. This file logs some but not all user activity. Because this is meant for use by Platfora, the format of this file is not guaranteed nor would you necessarily find all expected events in this log.
proton-detailedbuild.log An internal log file, this file contains messages from the lens build jobs.
proton-interestbuild.log An internal log file, this file contains messages from lens builds. Each entry is a set of comma-separated values in the following order:
  • user
  • sessionId
  • ipAddress
  • event
  • start
  • duration
  • buildrecordid
  • sources
  • reports
  • hostname
  • jobtracker
Because this is meant for use by Platfora, the format of this file is not guaranteed nor would you necessarily find all expected events in this log.
