Configuring Data Lineage Properties

Users with the System Administrator role can data lineage levels. Configure these levels in the System > Global Settings > Advanced > Lineage panel.

System administrators can edit the following global configuration properties on the System page:

Property

Default

platfora.lineage.max.levels.display

Specifies the maximum number of node levels (tree height) that Platfora includes in data lineage reports displayed in the web application. Each catalog object that is also an object parent represents a new node level. For example, a data lineage report that includes one lens computed field -> one lens base field -> lens -> dataset -> data source has four (4) levels (height of four).

5

platfora.lineage.max.levels.total

Specifies the maximum number of node levels (tree height) that Platfora fetches for data lineage reports. Platfora includes no more than this number of node levels per field when exporting a data lineage report to a JSON file. Each catalog object that is also an object parent represents a new node level. For example, a data lineage report that includes one lens computed field -> one lens base field -> lens -> dataset -> data source has four (4) levels (height of four).

10

platfora.lineage.max.files.display

Specifies the maximum number of source files specific to the lens build to list in a data lineage report that is displayed in the Platfora web application. When this value exceeds the value in platfora.lineage.max.hdfs.files.total, the data lineage report limits the number of files to platfora.lineage.max.hdfs.files.total. Enter a value of -1 to display all files.

100

platfora.lineage.max.files.total

Specifies maximum number of source files specific to the lens build Platfora fetches from HDFS for data lineage reports. Platfora includes no more than this number of files when exporting a data lineage report to a JSON file. Enter a value of -1 to fetch all files.

1000
