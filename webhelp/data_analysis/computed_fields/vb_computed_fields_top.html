Differences between Dataset and Vizboard Computed Fields

You can add computed fields in a dataset or in the vizboard. While computed fields are largely the same in both interfaces, there are differences you need to be aware of.

Raw Source versus Pre-aggregated Lenses

In a dataset, a computed field is built against the raw source data. When the build completes, the results is pre-aggregated data in a lens. Changing a computed field in a dataset, causes the lens to rebuild.

Computed fields in Vizboards run on the lens data. When you change a computed field in the Vizboard, the computed fields are immediately available for use in visualizations. There is no need to rebuild the lens.

Limitations of Vizboard Computed Fields

Because the vizboard can only query pre-aggregated lens data, vizboard computed fields have limitations that computed fields in datasets don't have. These limitations are the following:

  • You can only define a new vizboard computed field that operates on fields that exist in the currently selected lens. A vizboard computed field can break if it operates on fields that are later removed from the lens or dataset.

  • A vizboard computed field is only available within the vizboard where it was defined. It is not available from the dataset or other lenses.

  • You cannot use aggregate functions to add new measures from dimension data in the Vizboard. You can compute new measures from existing measures already in the lens. This means you can use ROLLUP function but you cannot use SUM or AVG.

  • You cannot use custom user-defined functions (UDFs) in a vizboard computed field expressions.

  • You cannot edit or delete a vizboard computed field if a visualization relies on it. Instead, remove the field from the visualization, then edit or delete it.
