Common Causes of Expression Errors

You cannot save a computed field that contains an error in the expression syntax. If your expression is not valid syntax, you will get an error message. This section describes some common expression errors and possible causes to investigate.

Error like this?

Platfora expected the string `)', but instead received `F'.
TO_LONG(New Field)

Possible causes:

  • Does a field need to be enclosed in square brackets (bracketed)? Field and dataset names used in a function must be enclosed in square brackets ([ ]) if they contain spaces, special characters, reserved keywords, or start with numeric characters.

  • Does a function use invalid syntax? Check the function reference documentation and verify the syntax of the function. Look for misspelled words or missing key words, arguments, brackets, or parentheses.

  • Does a field or dataset name contain a character that needs to be escaped? Field and dataset names may contain a right square bracket (]), but it must be preceded by another right square bracket (]]).

  • Does a literal string contain a character that needs to be escaped? Literal strings may contain a double quote ("), but it must be preceded by another double quote ("").

Error like this?

Field not found: carrier_name

Possible causes:

  • Does a literal string need to be enclosed in double quotes (")?

  • Is the field in a referenced dataset and need to be qualified with the reference name? Functions can use a field that is in a dataset referenced by the current dataset. You must specify the full path of the downstream field name by using the reference field name, like referencename.fieldname.

Error like this?

Function REGEX takes 2 arguments with types STRING, STRING, but one
argument of type INTEGER was provided.

Possible causes:

  • Verify all arguments use the correct data type.

  • If the argument is a field, you might need to change the data type in the base field or use a data type conversion function.

  • If the argument is an expression, edit the expression so it resolves to the proper data type.
