Advanced Properties

The advanced properties allow you to configure some advanced internal Platfora processes. Most properties should be changed only when instructed by customer support. Note that some advanced properties descriptions are included in their own sections.

  • platfora.admin.buildSummarySince
    The location on the local disk where Platfora is installed.
  • platfora.build.distinct.partitionsize
    Used to determine the number of distinct partitions for a lens build.
  • platfora.buildCleaner.force.threshold
    Enter the number of times the build cleaner waits for an opportunity to run before running, even if a lens is currently being built or queried against.
  • platfora.buildCleaner.interval
    Enter how much time to wait before the build cleaner runs again to remove unused lens files.
  • platfora.buildCleaner.purged.expiry
    Enter the amount of time in minutes the build cleaner waits before removing lens build files in HDFS that have already been unbuilt and deleted from Platfora's local disk.
  • platfora.catalog.cache.enabled
    Enables caching of catalog metadata.
  • platfora.cluster.status.timeout.millis
    Time until an unanswered cluster node ping is considered to indicate unavailability.
  • platfora.csv.download.max.rows
    Sets the maximum number of lens rows for a single download operation.
  • platfora.data.download
    Enables or disables data download of an entire lens.
  • platfora.data.export
    Enables or disables data export of an entire lens.
  • platfora.datasetparts.hive.limit
    Enter the number of Hive partitions/buckets to scan when creating a new dataset. You might want to limit this number to improve performance.
  • platfora.debug.packing.size
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.dfscache.prune
    Remove unknown files from the DFS cache on startup.
  • platfora.dfscache.size.fraction
    Enter the percentage (expressed as a fraction) of the local disk that Platfora uses to store lens build data in the Fractal Cache (DFS cache).
  • platfora.disable.temp.cleanup
    Choose whether or not Platfora should remove the temporary lens files on HDFS after a lens build completes.
  • platfora.distributed.query.exec.thresh
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.enable.temp.cache
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.filesystem.browse.timeout
    Enter the number of seconds before Platfora returns an error in the web application when trying to connect to an unresponsive data source during data ingest.
  • platfora.hive.direct.partitions
    When Platfora uses a JDBC connection to a Hive MetaStore database, choose whether or not Platfora fetches partition information directly from the database instead of using the client API.
  • platfora.hive.metastore.concurrency
    Enter the maximum number of concurrent connections Platfora makes to the Hive metastore.
  • platfora.jdbc.connectionTimeout
    Maximum time in seconds an idle JDBC connection is retained before closing.
  • platfora.jdbc.debugConnectionLeak
    Choose whether or not to capture a stacktrace in the Platfora server logs for leaked JDBC connections when the platfora.jdbc.connectionTimeout timeout is reached.
  • platfora.jdbc.logsql
    Choose whether or not to enable SQL logging.
  • platfora.jdbc.maxConnections
    Enter the maximum number of JDBC connections Platfora can make to the Metadata Catalog stored in a Postgres database.
  • platfora.jdbc.obtainConnectionTimeout
    Enter the maximum time in milliseconds Platfora tries to create a JDBC connection before throwing a SQL exception.
  • platfora.lineage.max.files.display
    The maximum number of source files specific to the lens build to list in a data lineage report that is displayed in the Platfora web application.
  • platfora.lineage.max.files.total
    The maximum number of source files specific to the lens build Platfora fetches from HDFS for data lineage reports.
  • platfora.lineage.max.levels.display
    The maximum height of the tree in the data lineage report represented by the number of node levels as displayed in the Platfora web application.
  • platfora.lineage.max.levels.total
    Enter the maximum height of the tree in the data lineage report represented by the number of node levels that Platfora fetches for data lineage reports.
  • platfora.local.coordinator.report.progress.millis
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.log.gc.duration.thresh
    Sets a threshold logging garbage collections.
  • platfora.max.worker.threads
    Maximum number of worker threads to use for query processing.
  • platfora.num.local.dfscache.prefetchers
    Enter the number of threads Platfora uses to copy lens files from HDFS to the Platfora cluster.
  • platfora.pool.array.fraction
    Percentage of available Java heap space consumed by currently unused temporary arrays.
  • platfora.pool.columnbuffer.fraction
    Percentage of the total Java heap space to use for caching lens data in memory.
  • platfora.pool.query.agg.memory.fraction
    Fraction of memory used for aggregation operation
  • platfora.query.always.sort
    Enables or disables query sorting.
  • platfora.query.assign.primary.workunits.threshold
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.query.logging.block.rows
    Enter the number of rows of intermediate query results to write to the Platfora server logs in debug mode. For Platfora internal use only.
  • platfora.query.parallelize.limit
    Enables or disable parallelize limit during query processing.
  • platfora.query.plan.runner.init.timeout.millis
    Enter the amount of time in milliseconds a query has to initialize before aborting the query.
  • platfora.query.plan.runner.timeout.millis
    Enter the time in milliseconds the query processor should wait before timing out.
  • platfora.query.starjoin
    Controls which query engine your installation uses.
  • platfora.query.timeout.millis
    Enter the amount of time in milliseconds a query is unresponsive before aborting the query.
  • platfora.query.use.master
    Choose whether or not some query tasks should only run worker nodes, or on worker nodes and the master node.
  • platfora.query.workunits.pct.per.request
    This is used to finetune the queries. Only change when instructed by Platfora engineering or Customer Support.
  • platfora.querynodes.per.block
    Replication factor for lens data.
  • platfora.rawdataset.cacheenabled
    Choose whether or not to enable the dataset cache.
  • platfora.rawdataset.cacheentryblocksizekb
    The block size in KB of each entry in the dataset cache.
  • platfora.rawdataset.cacheentrynumlimit
    Enter the maximum number of total entries (source files) Platfora stores in the dataset cache.
  • platfora.rawdataset.cacheexpirehours
    Enter the number of hours an entry (source file) is kept in the dataset cache before it expires.
  • platfora.rawdataset.cachesizemb
    Enter the maximum size in MB Platfora uses to store an entry (source file) in the dataset cache.
  • platfora.rawdataset.cachetimeoutminutes
    Enter the maximum time in minutes to allow for downloading remote source data during data ingest (which will be loaded into the dataset cache as a cache entry) before timing out.
  • platfora.rendering.enabled
    Allow vizboard-to-PDF rendering.
  • platfora.rendering.job.timeout
    Timeout (in seconds) for long-running jobs.
  • platfora.rendering.job.artifact.maxAge
    Number of days to retain job artifacts
  • platfora.rendering.maxJobs
    Maximum number of parallel rendering jobs.
  • platfora.requestcache.cachesize
    Define the maximum size of the HTTP request cache in bytes.
  • platfora.requestcache.objectsize
    Enter the maximum object size in bytes for objects stored in the HTTP request cache.
  • platfora.unicode.escaping.json
    Choose whether or not to escape Unicode characters in data values when generating visualizations.
  • platfora.unique.query.ids
    Choose whether or not to generate query IDs that are unique across restarts.
  • platfora.viz.max.rows
    Maximum number of result records (rows) to return for a visualization.
  • platfora.web.api.exception.included
    When enabled, any REST API call that results in an exception in Platfora includes the error stack trace in the body of the HTTP response.
  • platfora.workunit.size
    Number of blocks in each workunit assignment for the Query Processor.
