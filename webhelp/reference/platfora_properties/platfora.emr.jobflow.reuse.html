platfora.emr.jobflow.reuse

Choose whether or not to reuse existing job flows.

Full Description

Choose whether or not to reuse existing job flows. When enabled (set to true), job flows stay alive after a lens build finishes. The job flow terminates when it has been inactive for the configured timeout value (platfora.emr.shutdown.timeout). When starting a lens build, the server looks for an existing compatible job flow, and if found, it builds the lens using that job flow instead of launching a new one. A job flow is considered compatible when the following conditions are true:

  • It was launched from the same host using the same Platfora version.

  • It uses the same settings for instance count, instance types, Hadoop version, endpoint, placement, and bootstrap actions.

  • Either the AMI version is the same, or the Platfora server has "latest" as its AMI version.

Note that the log URI or termination protection settings are not used to determine compatibility.

When disabled (set to false), a separate EMR flow is launched for each lens build, and terminates when the build finishes.

Default

true (enabled)

Value Range/Type

Boolean (true, false)

Configurable in Web Application

Yes

Node Override

No

Restart Requirements

Requires Platfora server restart
