Amazon EMR and S3 Properties

The Amazon EMR and S3 properties allow you to configure how Platfora connects to AWS and initializes an EMR cluster. When EMR is enabled, Platfora initializes an EMR cluster for each lens build and (by default) terminates the cluster instances after the lens build has completed.
  • platfora.dfs.intermediatedir
    Specify a directory that Platfora uses to store intermediate lens build data.
  • platfora.emr.accesskey
    Enter your AWS account access key ID.
  • platfora.emr.ami.version
    The EMR Amazon Machine Image (AMI) to use to initialize EC2 instances.
  • platfora.emr.bootstrap.action
    Specify a bootstrap action to run on job flow startup.
  • platfora.emr.ec2key.name
    Name of the EC2 key pair to use for SSH access to your EMR instances.
  • platfora.emr.endpoint
    The Amazon EMR endpoint for your region (for example, elasticmapreduce.us-west-1.amazonaws.com).
  • platfora.emr.hadoop.version
    The Hadoop version to use for EMR.
  • platfora.emr.instance.count
    Number of EC2 instances to initialize for a Platfora lens build.
  • platfora.emr.instancetype.master
    Specify the EC2 instance type to initialize for the EMR Hadoop JobTracker node.
  • platfora.emr.instancetype.slave
    Specify the EC2 instance type to initialize for the EMR Hadoop TaskTracker nodes.
  • platfora.emr.jardir
    Enter an S3 directory where Platfora writes its jar files.
  • platfora.emr.jobconfdir
    The S3 directory location where Platfora writes its build metadata files.
  • platfora.emr.jobflow.reuse
    Choose whether or not to reuse existing job flows.
  • platfora.emr.log.uri
    Specify an S3 Native URI where EMR periodically synchronizes its job flow log files.
  • platfora.emr.placement
    The region plus availability zone code that is available for your Amazon Web Services (AWS) account (for example: us-west-2a).
  • platfora.emr.secretkey
    Enter your AWS account secret key.
  • platfora.emr.shutdown.runway
    When platfora.emr.jobflow.reuse is enabled (set to true), this configures the amount of time in seconds to begin terminating a job flow before the next EMR payment increment occurs.
  • platfora.emr.shutdown.timeout
    Job flow inactivity timeout in seconds.
  • platfora.emr.status.timeout
    When platfora.emr.jobflow.reuse is enabled (set to true), this configures the time in seconds to wait for a response from the EMR job flow before terminating it.
  • platfora.emr.termination.protected
    Choose whether or not EMR instances are terminated after a lens build.
  • platfora.emr.visibility.allusers
    Choose whether or not to make the EMR job flow visible to all users.
  • platfora.S3.accesskey
    Enter your Amazon S3 account access key ID.
  • platfora.s3.endpoint
    The endpoint Platfora uses to access S3 buckets.
  • platfora.s3.filesystem.optimization.enable
    Choose whether or not to enable Platfora's optimized S3 access library.
  • platfora.S3.secretkey
    Enter your Amazon S3 secret key.
