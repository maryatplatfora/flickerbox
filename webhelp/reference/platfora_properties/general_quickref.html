Bootstrap Configuration Properties

The installation process collects several bootstrap configuration properties. These values appear in Platfora's Global Settings as read-only values.

List of Bootstrap Configuration Properties

The table below lists the bootstrap configuration properties:

Property

Default

Description

platfora.jdbc.connection

jdbc:postgresql://localhost/platfora

JDBC connection URL for metadata catalog. JDBC connection string for the database where Platfora stores its metadata in Postgres.

platfora.webservice.port

8001

The port used to access the Platfora GUI with a web browser using HTTP.

platfora.server.management.port

8002

The port over which the Platfora server and worker nodes communicate with each other.

platfora.home

Determined during installation.

The location on the local disk where Platfora is installed.

platfora.data.dir

Determined during installation.

Local directory Platfora uses to store source from lens builds and various metadata and log files. Only used when $PLATFORA_DATA_DIR is not set.

platfora.support.identifier

Determined during installation.

Enter an identifier, such as your organization name, that will be used in syscapture file names. Choose a string that Platfora Customer Support can easily associate with your organization.

platfora.dfs.datadir

${platfora.dfs.datadir}

When Platfora builds lenses, it stores data in memory and in the configured Hadoop DFS. Specify a DFS data directory where Platfora stores its data in Hadoop. When the data directory is on a slow disk (such as S3), Platfora builds make use if the platfora.dfs.intermediatedatadir to improve performance.

platfora.multinode.nodeId

Determined during installation.

ID number of the current Platfora node.

platfora.dfs.libdir

$PLATFORA_DATA_DIR/lib

Displays the Hadoop DFS directory where Platfora stores its jar files. This value is set by Platfora's installation. You cannot set it through the application client.

Changing Bootstrap Properties

The Platfora installation process stores the values of these bootstrap configuration properties in Platfora's metadata store. To change these values, you need to re-run the setup.py utility. After a quick check, the system prompts you to change the bootstrap values:

...
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-= SUMMARY =-=-=-=-=-=-=-==-=-=-=-=-=-=-=
Platfora Configuration Directory: /home/platfora/platfora_conf
Hadoop Version: pivotal_1.1
Platfora Web Services Port: 8001
Platfora Server Management Port: 8002
Enable Platfora Application for secure connection: false
Hadoop Configuration Directory: /home/platfora/hadoop_conf
Local Data Directory: /data/platfora_data
Platfora JVM Size: 8G
Company name for support diagnostics: maryco
Platfora DFS Cache size (as fraction of total disk capacity): 0.8
=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=-=

Would you like to make any configuration changes (Y/N)? (No)

You can only change the bootstrap properties when the server is stopped.
