Understand the Data Pipeline

When your system administrator installs Platfora, the setup process connects the Platfora server to your Hadoop cluster. Once in Platfora, users that are system administrators, data administrators, and analysts work within Platfora's data pipeline. The following diagram depicts the four steps of the Platfora data pipeline.

The following sections discuss each step in greater detail.

Step 1: Connect to Data Sources

Platfora relies on connections to a data source. A connection can be a mount point or directory on an external data provider. Platfora supports many data providers. You can also add your own custom connectors. Some connectors Platfora supports out of the box are:

  • HDFS

  • HIVE Metastore

  • MapR FS

  • Amazon S3

Data files can contain data on web logs, telementry, customer demographics and much, much more. Platfora provides built-in support for a number of source file formats, including Hive table definitions, delimited, Avro, JSON, XML, and web access logs.

Only users that have System Administrator privileges can create data source connections. System administrators also secure these data sources. Platfora users can only interact with data they are authorized for.

Step 2: Create the Datasets

A dataset is a metadata description of how the data is structured. Only users with Data Administrator privileges can create datasets. When you create a dataset you can:

  • set parsing logic

  • manage field format and descriptions

  • write expressions or formulas

  • set aggregation of measures

  • define dataset keys

  • reference or join datasets

  • create metadata describing the contents

If you are familiar with extract, transformation, and load (ETL) workflows, the creation of a dataset encompasses the extract and transform logic.

Step 3: Create a Lens

A lens is a self-service view of a dataset. Any user with Analyst privileges can add or build a lens. To create a lens, the user first selects the data fields of interest from a dataset. When the user is done, Platfora populates the lens' fields by launching a lens build.

A lens build automates a number of Hadoop processing tasks. The build submits a series of MapReduce jobs to Hadoop, collects the results, and brings the results back into Platfora. The data populated to a lens is pre-aggregated, compressed, and columnar. From the perspective of an ETL workflow, the lens build would be the "load" part of the process.

Datasets and the lenses built from them together comprise a data catalog. In the data catalog, business users can browse both lenses or datasets. Users can suggest new lenses based on datasets. Or users with the appropriate privilege can build their lenses to analyze or to share with others.

Step 4: Analyze Data using the Vizboard

Once a lens is built, the data is available to build a visualization. A visualization is the visible result of a data analysis. The Vizboard is Platfora's data analysis tool.

A user selects a lens to analyze from the Vizboard. By manipulating and filtering data fields, the analyst searches for data patterns or correlations within the data. Visualizations can take various forms such as bar charts, scatterplots, and heatmaps.

Users can build up a dashboard of multiple visualizations. For example, a vizboard may show two or three charts that support a future business direction or confirm a results of past decisions.
