The Platfora Architecture

Platfora runs on dedicated servers in the same network as your Hadoop cluster, and uses the Distributed File System (DFS) and MapReduce services of Hadoop. The Platfora server architecture is comprised of three main components: the Lens Builder, the In-Memory Data Server, and the Platfora Application Server.


The Lens Builder

The lens builder hides Hadoop's complex processing and query framework from users. Users request data from the Platfora data catalog. Platfora's lens builder translates the request into a series of custom MapReduce jobs. Then, the builder submits the MapReduce jobs to Hadoop.

MapReduce processes, transforms, and aggregates the requested data. Platfora retrieves the job's results from DFS and places it in its on-disk storage. Then, Platfora makes the results available to users as an optimized data structure called a lens.

Business analysts use a lens to create a visualization in Platfora.

In-memory Data Server

Visualizations are the visible results of a user's data analysis. When users construct visualizations, they choose a lens to work with. Choosing a lens loads its data into Platfora's in-memory data server. Users analyse the data by selecting fields, filtering them, and placing them on graphical drop zones to develop a visualization.

This user activity results in underlying queries against the data. Platfora's in-memory data server manages building the queries, running them, and returning their results. All users experience are the visible changes to their visualization. For example, a user might filter lens data by a single field and a heatmap shows increased intensity as a result.

As users work with lens data the server optimizes the queries required to render the visualization that the user wants. Administrators can add in-memory server nodes to scale up lens capacity and balance query workloads.

Platfora Application Server

Platfora's user interface runs as a web application in your network. Users connect to Platfora using any HTML5-compliant browser. Through the browser, users interact with data in Hadoop as easily as browsing a web site. Metadata for the Platfora application is stored in a relational database managed by the Platfora master server.
