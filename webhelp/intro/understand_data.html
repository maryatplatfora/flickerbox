Understand Dimensions and Measures

This section introduces the concept of data dimensions and measures. It is important to understand these concepts as they can impact lens build design and efficiency.
At the dataset level and later at the lens build level, Platfora distinguishes between dimension data and measure data. Either type of data can appear in a dataset or lens field.

Data Dimensions

A dimension is data that answers who, what, where, and when questions. The following table shows examples of each.

Dimension Type

Example

Who

customer name, gender, title, customer ID, social security number

What

product type, call type, account type, product ID, revenue, orders, sales

Where

geo code, zip code, sales channel id, address, state, longitude, latitude

When

date, timestamp, month

Dimensions can be numeric, text, or time-based. You can categorize or group dimension data. If you are familiar with SQL, dimensions are values you would GROUP BY. For example, all the men in the 94403 zip code or total sales by month.

Data Measures

A measure is a numeric value representing an aggregation. Measures answer "how" questions about data such as how many or how long? If you are familiar with SQL, measure values come from the aggregate functions such as SUM(), COUNT(), MAX(), MIN(). The following table shows examples of measures:

Measure Type

Example

How much or how many

sum of all revenue by geo, sum of all sales, count distinct numaximum unit price, minimum number of feedback

How long

total minutes on a call, number of days between sales
To what degree maximum number of users, minimum page hits

You can apply a measure to any field. For example, you might sum the sale column to get the total sales (784.92). You also might count the distinct products in the Product column (3). Even though one Sale is an numeric and the Product is text, the resulting measure in both cases is a numeric.

Not all dimensions lend themselves to all measures. For example, you could sum the Prod_id column but it wouldn't make much sense to total the product ids. It would make sense to maybe count how many distinct products you have.
